Abstract

Provenance as prerequisite to AGI-scale belief

Large language models propagate ideas without lineage: Confucius is said to have written the Dao De Jing; Socrates is treated as author of Plato's Republic; Freud is credited with cognitive dissonance; nirvana becomes "eternal heaven." Sophia AGI (σοφία, wisdom) is an open corpus, per-domain benchmark, and three-path agent that enforces source discipline — evidence about authorship, tradition, and subfield before reasoning. This document presents the thesis-level design of the project and the council-mediated web interface that exposes it.

Chapter I

The lineage-merge failure mode

Epistemic harm in LLM outputs is not only factual error but attribution collapse: distinct intellectual traditions are merged into a single undifferentiated voice. The failure is structural. When a model answers "Did Confucius write the Dao De Jing?" without denying the trap, it licenses centuries of conflation between 儒家 and 道家 registers.

Sophia treats each trap as measurable. Philosophy benchmarks deny false authorship; psychology benchmarks separate cognitive, clinical, and pop_myth subfields; religion benchmarks require council panel answers that seat theological, historical-critical, and comparative voices without merger.

Chapter II

Source discipline as theoretical framework

Source discipline is the project's core construct (see docs/08-Domains/Source-Discipline-Methodology.md). It requires five operations on every answer:

  1. Named attribution — attributedAuthor, doNotAttributeTo in data records
  2. Confidence signaling — compiled, legendary, disputed, consensus
  3. Boundary maintenance — traditions and subfields must not merge
  4. Bilingual anchoring — canonical 中文 terms + 中文摘要
  5. Hub examples — one training pair may cover multiple benchmark traps (e.g. example 001 → four philosophy cases)

The framework generalizes from philosophy to psychology and religion centers without changing the epistemic contract: retrieval of records precedes generation; a gate checks discipline markers before release.

Chapter III

Methodology: corpus, benchmark, gate

3.1 Data layer

Structured JSON in data/: attributions.json, psychology_concepts.json, religion_concepts.json, traditions.json. 515 bilingual training examples in training/examples/ export to corpus.jsonl on Hugging Face.

3.2 Benchmark layer

Per-domain cases in tests/benchmark-{domain}.json. Scoring via tools/score_benchmark.py uses explicit markers: denial patterns, myth labels, council format, tradition ids, subfield tags. Reference teacher responses score 100% on all domains; external models are compared on the same harness.

3.3 Agent layer (v0.6.0)

Three paths — advisor, repo, life — share curated RAG (rag/index), Claude or Gemini via configured API, epistemic self-check, and decision memory (agent/memory/decisions.jsonl). Local LoRA sophia-v1 (Qwen2.5-3B) scores 20/23 (87%) on the benchmark harness.

Chapter IV

UI Council: how this website was decided

Following religion council mode, all design voices sit on one panel. No single aesthetic wins by default; tensions are named. Full record: docs/10-Web/UI-Council-Decisions.md.

Council panel (all seated): UX Research · Design Systems · Accessibility · Engineering · Philosophy lineage
UX Research voice
Thesis-first information architecture: Abstract through Agent, not marketing funnel. Table of contents persistent; chapters scannable.
Design Systems voice
Scholarly monograph aesthetic: ivory paper, ink type, bronze accent. Serif body, sans chrome. Council panels visually echo religion answer format.
Accessibility voice
18px base, 1.65 leading, skip link, focus states, text labels on all scores. CJK-friendly font stack.
Engineering voice
Static web/ for GitHub Pages; tools/serve_web.py adds /api/ask for live agent. Manifest regenerated by build_web_data.py.
Philosophy lineage voice
Site must practice source discipline: cite paths, show benchmark evidence, tagline Wisdom before intelligence without hype.
Debate / tension: Thesis depth vs. mobile brevity → progressive disclosure: full chapters on desktop; collapsed nav on small screens; agent panel optional.

Chapter VII

Online RAG: Gemini API & Vertex AI

v0.6.0 adds curated online RAG — no open-web grounding. Retrieval uses a pre-built index (rag/index/chunks.jsonl, benchmark holdouts excluded) over data, disputes, domain docs, reference answers, and teacher examples. Generation runs on Gemini API (local MVP) or Vertex AI (Cloud Run production), with Claude fallback and the same epistemic gate as the agent CLI.
